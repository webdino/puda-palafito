# Google Drive API の有効化と設定手順

Puda Palafito 拡張機能で Google Drive への自動保存機能を利用するためには、Google Cloud Console にて「Google Drive API」を有効化し、専用の認証情報（OAuth クライアント ID）を作成する必要があります。

以下の手順に沿って設定を進めてください。

---

## ステップ 1: 新しいプロジェクトの作成

1. [Google Cloud Console](https://console.cloud.google.com/) にアクセスし、Google アカウントでログインします。
2. 画面左上にある「**プロジェクトの選択**」プルダウン（すでにプロジェクトがある場合はそのプロジェクト名）をクリックします。
3. ポップアップ画面の右上にある「**新しいプロジェクト**」をクリックします。
4. **プロジェクト名**を入力します。（例: `puda-palafito-ext`）
5. 「**作成**」ボタンをクリックします。作成が完了するまで数秒〜数十秒待ちます。
6. 作成が完了したら、通知アイコン（ベルマーク）などから先ほど作ったプロジェクトを「**選択**」して、そのプロジェクトの画面に入ります。

---

## ステップ 2: Google Drive API の有効化

1. プロジェクトを選択した状態で、画面左側のナビゲーションメニュー（三本線アイコン）を開き、**「API とサービス」 > 「ライブラリ」** をクリックします。
2. 検索バーに `Google Drive API` と入力して検索します。
3. 検索結果から **「Google Drive API」** をクリックして詳細ページを開きます。
4. 青い **「有効にする」** ボタンをクリックします。

---

## ステップ 3: OAuth 同意画面の設定（初めて作成する場合のみ）

APIを有効にしたら、次にユーザー認証画面（「ログインして連携」を押した時に出る画面）の基礎設定を行います。

1. 再び左側のメニューから **「API とサービス」 > 「OAuth 同意画面」** をクリックします。
2. **User Type** の選択肢が出ます。社内・組織向けでない場合は **「外部 (External)」** を選び、「**作成**」をクリックします。
3. 以下の必須項目（赤いアスタリスク）を入力します。それ以外は空欄で構いません。
   - **アプリ名**: 例 `Puda Palafito`
   - **ユーザーサポートメール**: あなたのメールアドレスを選択
   - **デベロッパーの連絡先情報**: あなたのメールアドレスを入力
4. 入力後、画面一番下の「**保存して次へ**」をクリックします。
5. **「スコープ（Scopes）」** の画面になります。
   - 「**スコープを追加または削除**」ボタンをクリックします。
   - 「フィルタ」という検索バーに `drive.file` と入力します。
   - `https://www.googleapis.com/auth/drive.file` （Google ドライブ内の特定のファイルのみ作成、表示、編集など）の左側チェックボックスにチェックを入れます。
   - スクロールして「更新」を押し、画面一番下の「**保存して次へ**」をクリックします。
6. **「テストユーザー」** の画面になります。
   - 右側にある「**+ ADD USERS**」をクリックして、今回ご自身や想定されるテストユーザーのGoogleアカウントのメールアドレスを入力します。
   - 入力したら「追加」を押し、画面一番下の「**保存して次へ**」をクリックします。
7. **「概要」** 画面が表示されたら、「ダッシュボードに戻る」を押します。

> **参考**: 同意画面が「テスト（Testing）」状態の場合、追加した「テストユーザー」しかこのログイン機能を使えません。本番公開したい場合は後ほど「アプリを公開」ボタンを押しますが、その場合はGoogleの審査が必要になります。

---

## ステップ 4: OAuth クライアント ID の作成と取得

同意画面の設定が終わったら、最後にChrome拡張機能からアクセスするための「ID」を発行します。

1. 左側のメニューから **「API とサービス」 > 「認証情報」** をクリックします。
2. 画面上部にある **「＋認証情報を作成」** をクリックし、**「OAuth クライアント ID」** を選びます。
3. **「アプリケーションの種類」** をクリックし、**「Chrome アプリ」** を選択します。
4. 以下の項目を入力します。
   - **名前**: 任意の名前（例: `Puda Chrome Client`）
   - **アイテム ID**: 実機に読み込ませた拡張機能の ID を入力します。（※注意参照）
5. 「**作成**」をクリックします。
6. 画面に **クライアント ID**（長めの文字列）が表示されるので、それをコピーします。

> **※拡張機能の「アイテム ID」の確認方法**
> 1. プロジェクトをビルド（`pnpm build`）してChromeに拡張機能を読み込みます。（`chrome://extensions/` 画面から「パッケージ化されていない拡張機能を読み込む」で `.output/chrome-mv3` を選択）
> 2. その一覧画面にある拡張機能の右下に書かれている「**ID:** `xxxxxxxxxxxxxxxxxxxxxxxx`」というアルファベットの文字列がアイテムIDです。

---

## ステップ 5: 環境変数 (.env) への設定

1. 先ほどコピーした **クライアント ID** を、ローカルプロジェクトの `.env` ファイルに貼り付けます。
   
   ```env
   WXT_OAUTH_CLIENT_ID=ここに先ほどコピーしたクライアントIDを貼り付けます
   ```

2. ファイルを保存し、一度 `pnpm dev` や `open-chrome-dev.bat` を再起動します。
3. 拡張機能の Options 画面の下部にある「Google アカウントでログイン」を押すと、正常に認証画面が立ち上がるはずです。
