# Google Drive API の有効化と設定手順

Puda Palafito 拡張機能で Google Drive への自動保存機能を利用するためには、Google Cloud Console にて「Google Drive API」を有効化し、専用の認証情報（OAuth クライアント ID）を作成する必要があります。

以下の手順に沿って設定を進めてください。

---

## ステップ 1: Google Cloud 新しいプロジェクトの作成手順

### 1. Google Cloud Console へのアクセス

1. [Google Cloud Console](https://console.cloud.google.com/) にアクセスし、Google アカウントでログインします。

### 2. プロジェクト選択メニューを開く

1. 画面上部（「Google Cloud」ロゴの右側）にある現在のプロジェクト名が表示されているドロップダウンをクリックします。
2. プロジェクトの選択ウィンドウが表示されます。

### 3. プロジェクト作成画面へ移動

1. ウィンドウの右上にある **[新しいプロジェクト]** をクリックします。

### 4. プロジェクト情報の入力

1. **プロジェクト名**: 任意のプロジェクト名を入力します（例: `puda-palette-ext`）。
    * 下部に表示される「プロジェクト ID」は、プロジェクト名に基づいて自動生成されますが、必要に応じて編集可能です（作成後は変更できません）。
2. **場所**: 組織に属している場合は「親組織」や「フォルダ」を選択します。個人アカウントの場合は「組織なし」のままで問題ありません。

### 5. プロジェクトの作成と確認

1. **[作成]** ボタンをクリックします。
2. 画面右上の通知アイコン（ベルのマーク）に「プロジェクトを作成しています」と表示されます。
3. 作成が完了すると通知が「プロジェクトを作成しました」に変わり、自動的に新しいプロジェクトのダッシュボードへ切り替わるか、再度プロジェクト選択メニューから新しいプロジェクトを選択できるようになります。

## ステップ 2: Google Drive API の有効化

### 1. API ライブラリを開く

1. [Google Cloud Console](https://console.cloud.google.com/) のナビゲーションメニュー（左上の三本線アイコン）をクリックします。
2. **[API とサービス]** > **[ライブラリ]** を選択します。

### 2. Google Drive API を検索する

1. 「API とサービスを検索」ボックスに `Google Drive API` と入力し、Enterキーを押します。
2. 検索結果から **[Google Drive API]** をクリックします。

### 3. API を有効化する

1. Google Drive API の詳細画面が表示されたら、 **[有効にする]** ボタンをクリックします。
2. 有効化処理が開始されます。完了するまで数秒〜数十秒待ちます。

### 4. 有効化の確認

1. 自動的に「API とサービスの詳細」画面（ダッシュボード）に切り替わります。
2. 「ステータス：有効」と表示されていれば、プロジェクトで Google Drive API が利用可能な状態です。

## ステップ 3: Google Cloud OAuth同意画面の構成手順

このドキュメントでは、Google Cloudプロジェクトにおいて、アプリがGoogleユーザーのデータにアクセスするための「OAuth同意画面」を構成する手順について説明します。

### 1. OAuth同意画面の開始

1. Google Cloud Consoleのナビゲーションメニューから **[APIとサービス]** > **[OAuth同意画面]** を選択します。
2. 「Google Auth プラットフォームは構成されていません」というメッセージが表示されるので、 **[開始]** ボタンをクリックします。

### 2. アプリ情報の入力（ステップ1）

「プロジェクト構成」画面で、以下の情報を入力・選択します。

* **アプリ名**: 任意の名前を入力（例: `Puda Palafito`）。
* **ユーザーサポートメール**: 管理者のメールアドレスを入力。
* 入力が完了したら **[次へ]** をクリックします。

### 3. 公開範囲の設定（ステップ2）

「対象」セクションで、アプリの公開範囲を選択します。

* **外部**を選択します（組織外のGoogleアカウントを持つユーザーも利用可能になります）。
* 選択後、 **[次へ]** をクリックします。

### 4. 連絡先情報の入力（ステップ3）

「連絡先情報」セクションで、Googleからの重要な連絡を受け取るためのアドレスを入力します。

* **メールアドレス**: 管理者のメールアドレスを入力。
* 入力後、 **[次へ]** をクリックします。

### 5. 構成の完了（ステップ4）

1. 「終了」セクションで、内容を確認します。
2. 「Google API サービス：ユーザーデータに関するポリシー」への同意チェックボックスにチェックを入れます。
3. **[続行]** ボタンをクリックします。

### 6. 設定の確認

構成が完了すると、自動的に「OAuthの概要」ページに遷移します。

* 「OAuth の概要を作成しました」という通知が表示されます。
* 同意画面の設定内容（アプリ名やユーザータイプなど）が保存され、編集可能な状態になります。

## ステップ 4: OAuth クライアント ID の作成と取得

### 1. API とサービス画面への移動

1. Google Cloud Console の左上にある**ナビゲーションメニュー（三本線アイコン）**をクリックします。
2. **「API とサービス」** ＞ **「認証情報」** を選択します。

### 2. クライアント ID 作成画面の表示

1. 画面上部の **「+ 認証情報を作成」** ボタンをクリックします。
2. プルダウンメニューから **「OAuth クライアント ID」** を選択します。

### 3. クライアント ID の設定

1. **アプリケーションの種類**:
   * プルダウンから **「Chrome 拡張機能」** を選択します。
2. **名前**:
   * 任意の名前を入力します（例: `puda-palafito-ext`）。
3. **アイテム ID**:
   * ローカルで読み込んだ拡張機能の **ID** を入力します。

### 4. 設定の保存

1. 入力内容を確認し、画面下部の **「作成」** ボタンをクリックします。

### 5. クライアント ID の取得

1. 「OAuth クライアントを作成しました」というダイアログが表示されます。
2. 表示された **「クライアント ID」** （文字列）をコピーして保存するか、**「JSON をダウンロード」** をクリックして設定ファイルを保存します。
3. **「OK」** を押して完了します。

#### 補足事項

* **アイテム ID の確認方法**:

Chrome で `chrome://extensions` を開き、ロード済みの拡張機能の詳細を開いてください。  
その中に「ID」という項目があるので、その値をコピーしてください。

---

## ステップ 5: 環境変数 (.env) への設定

1. 先ほどコピーした **クライアント ID** を、プロジェクトの `.env` ファイルに貼り付けます。

   ```env
   WXT_OAUTH_CLIENT_ID=ここに先ほどコピーしたクライアントIDを貼り付けます
   ```
